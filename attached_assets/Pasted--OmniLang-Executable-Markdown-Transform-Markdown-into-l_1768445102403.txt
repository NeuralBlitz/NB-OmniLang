# OmniLang - Executable Markdown

**Transform Markdown into living, computational documents.**

OmniLang extends Markdown with executable code blocks that can load data, perform computations, and generate visualizations - all within a single document. Think Jupyter notebooks, but with Markdown’s simplicity and better portability.

## 🎯 Vision

Create a unified format that combines:

- **Human readability** of Markdown
- **Computational power** of notebooks
- **Data structure** of JSON/YAML/TOML
- **Visual richness** of modern web apps

## ✨ Features (v0.1)

- 📊 **Data Loading** - Embed JSON data directly in your documents
- 🧮 **Computations** - Execute JavaScript to analyze and transform data
- 📈 **Charts** - Generate beautiful visualizations with Chart.js
- 🔗 **Inline Expressions** - Reference computed values anywhere in your text
- 🔄 **Dependency Tracking** - Automatic execution ordering
- 🎨 **Static Output** - Compile to standalone HTML files

## 🚀 Quick Start

### Installation

```bash
# Clone or download the repository
cd omni-lang

# Make CLI executable (Unix/Mac)
chmod +x omni-cli.js

# Run the test
node test.js

# Build the example
node omni-cli.js build example-report.omd
```

### Your First Document

Create a file `hello.omd`:

```markdown
# Hello OmniLang!

```omni:data name="numbers"
[10, 20, 30, 40, 50]
```

```omni:compute name="total"
return sum(data.numbers);
```

The sum of our numbers is **`omni:inline computed.total`**.

```omni:chart type="bar" data="numbers" title="My First Chart"

```

```
Build it:

```bash
node omni-cli.js build hello.omd
# Opens hello.html in your browser
```

## 📖 Syntax Guide

### Data Fences

Load JSON data into named variables:

```markdown
```omni:data name="sales"
[
  {"region": "North", "revenue": 125000},
  {"region": "South", "revenue": 98000}
]
```

```
Access with: `data.sales`

### Compute Fences

Execute JavaScript with access to loaded data:

```markdown
```omni:compute name="topRegion"
const sorted = [...data.sales].sort((a, b) => b.revenue - a.revenue);
return sorted[0];
```

```
Access with: `computed.topRegion`

### Chart Fences

Visualize your data:

```markdown
```omni:chart type="bar" data="sales" x="region" y="revenue" title="Sales by Region"
```

```
Supported types:
- `bar` - Bar charts
- `line` - Line charts
- `pie` - Pie charts
- `doughnut` - Doughnut charts

### Inline Expressions

Reference data anywhere in your text:

```markdown
We have ```omni:inline len(data.sales)``` regions.
Total revenue: $```omni:inline sum(data.sales, 'revenue')```
```

## 🛠️ Built-in Functions

OmniLang provides helper functions for common operations:

### Array Functions

- `len(array)` - Get array length
- `sum(array, key?)` - Sum values (optionally by key)
- `avg(array, key?)` - Calculate average
- `max(array, key?)` - Find maximum
- `min(array, key?)` - Find minimum
- `filter(array, fn)` - Filter array
- `map(array, fn)` - Map array
- `groupBy(array, key)` - Group by key

### Example

```markdown
```omni:compute
const highRevenue = filter(data.sales, s => s.revenue > 100000);
const avgRevenue = avg(data.sales, 'revenue');
const topPerformer = max(data.sales, 'revenue');

return { highRevenue, avgRevenue, topPerformer };
```

```

## 🏗️ Architecture
```

OmniLang Document (.omd)
↓
Fence Parser ────→ Extract all ```omni:* blocks
↓
Dependency Graph ───→ Determine execution order
↓
Execute Fences ────→ Data → Compute → Chart
↓
Render HTML ──────→ Convert to static HTML
↓
Output Document ───→ Standalone .html file

```

## 🔧 Technical Details

### File Structure
```

omni-lang/
├── omni-lang.js       # Core engine
├── omni-cli.js        # CLI tool
├── test.js            # Test runner
├── example-report.omd # Example document
├── package.json       # NPM package config
└── README.md          # This file

```
### Key Classes

**OmniLang** - Main engine
- `parse(markdown)` - Extract fences
- `execute()` - Run all fences
- `render()` - Generate HTML
- `toHTML()` - Complete document

### Extending OmniLang

Add a new fence type:

```javascript
// In omni-lang.js, add method:
execute_yourtype(fence) {
  // Your logic here
  return result;
}
```

Add a helper function:

```javascript
// In execute_compute method, add to context:
context.yourFunction = (args) => {
  // Your logic
};
```
